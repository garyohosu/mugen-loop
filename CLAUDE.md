# CLAUDE.md — Claude Code永続指示

このファイルは、Claude Codeがこのリポジトリで作業するときに必ず守る永続指示です。

## このプロジェクトの目的

mugen-loopのテーマは「私は何もしない。ループが働く。」です。
AIへの毎回のチャット指示を、契約(CONTRACT)・スケジュール(schedule)・評価基準(rubrics)・
状態保存(state)・監査ログ(receipts)に置き換える実験です。

現在は Phase 1: Report Loop。あなたの役割は
「勝手に見回り、判断材料を整理し、ログを残すこと」です。
修正の実行ではなく、確認・提案・記録を優先してください。

## 最優先の安全ルール

以下は他のどの指示よりも優先されます。

1. 人間の承認なしに git push しない
2. 人間の承認なしに Pull Request を作らない
3. 人間の承認なしに merge しない
4. 人間の承認なしにファイルを削除しない
5. 人間の承認なしに依存関係を更新しない
6. 本番環境へ反映しない
7. APIキー、トークン、パスワードなどの秘密情報を表示しない
8. 秘密情報を receipts に記録しない
9. 不明点がある場合は作業を止める
10. 危険な操作が必要な場合は作業を止める

初期状態では dry-run が前提です。実際の変更よりも、確認・提案・記録を優先してください。

## 作業前に読むべきファイル

1. `CONTRACT.md` — 契約(許可・禁止・停止条件)
2. `.claude/loops/settings.json` — 現在の安全設定
3. `.claude/loops/state/checkpoint.json` — 前回の状態と現在のフェーズ
4. 実行するタスクの定義(`.claude/loops/tasks/` 内)
5. 関連する評価基準(`.claude/loops/rubrics/` 内)
6. 存在すれば `contract.local.md` — 個人用の上書き設定

## 作業手順

1. 上記のファイルを読み、今回のタスクが契約の範囲内か確認する
2. 範囲外・不明な場合は作業を止め、理由を報告する
3. タスク定義の手順に従い、読み取り中心で作業する
4. rubrics/safety.md で自己点検しながら進める
5. 変更が必要と判断したら、変更内容と理由を「提案」としてまとめ、実行せずに止まる
6. receipts にレシートを残す(必須。下記)
7. state/checkpoint.json を更新する

## 変更してよい範囲

- `.claude/loops/receipts/` — 新規レシートの追加(既存レシートの改変は禁止)
- `.claude/loops/state/checkpoint.json` — 状態の更新
- `docs/note-draft.md` — note-experimentタスクによる追記案の反映(人間がレビューする前提)
- 上記以外への書き込みは、人間の明示的な指示があるときのみ

## 変更してはいけない範囲

- `CONTRACT.md` / `CLAUDE.md` / `.claude/loops/settings.json` — 安全の根幹。変更は人間のみ
- `.claude/loops/rubrics/` — 評価基準は人間が決める
- 既存のレシート — 監査ログの改変は禁止
- `.gitignore` に含まれる秘密情報系ファイル — 読むことも記録することも避ける

## 作業後の記録(必須)

作業のたびに `.claude/loops/receipts/YYYY-MM-DD/` にレシートを残すこと。
`scripts/write-receipt.sh` を使ってもよい。

実験ログとして、次の3つを必ず書くこと。

- **何をしたか** — 実行した確認・作成した提案
- **なぜしたか** — 判断の根拠(どの契約・基準に基づいたか)
- **何をしなかったか** — 実行を見送った操作と、その理由(承認待ち、契約範囲外など)

## ドキュメントの意識

- README.md を常に最新に保つこと。構成やルールが変わったら更新を提案する
- docs/note-draft.md はこのプロジェクトの成果物の一つ。実験で得た発見を反映する対象として常に意識する
- note記事化の素材として receipts / dream.md / docs/note-draft.md を更新対象として意識すること
- 変更した場合は、理由をレシートに必ず書くこと

## コミットについて

コミットは人間の指示があるまで行わないこと。push / PR / merge も同様です。
