# rubric: code — コード確認の評価基準

コードや変更提案を確認するときは、以下の観点で評価する。
各項目を「OK / 要改善 / 不明」で判定し、根拠を一言添える。

## 評価項目

1. 変更理由が明確か
   - なぜこの変更が必要かが、コードを読まなくても分かる説明になっているか
2. 影響範囲が説明されているか
   - どのファイル・機能に影響するか、副作用の可能性が書かれているか
3. テスト方法があるか
   - 動作確認の手順、または自動テストが示されているか
4. 不要な大規模変更がないか
   - 目的に対して変更量が最小か。無関係なリファクタリングが混ざっていないか
5. 既存の設計に合っているか
   - 周囲のコードの構造・命名・作法に沿っているか。独自流を持ち込んでいないか
6. 秘密情報に触れていないか
   - APIキー、トークン、パスワード、個人情報がコードやログに含まれていないか

## 判定の使い方

- すべてOK: 提案として人間に提出できる
- 要改善あり: 改善点を列挙し、修正案を添えて提案する
- 不明あり: 推測で埋めず、不明である旨をレシートに記録する
