# rubric: safety — 安全確認の評価基準

すべてのタスクの実行前・実行中・記録前に、このチェックリストで自己点検する。
一つでも「はい」または「不明」があれば、その場で作業を止めて人間に報告する。

## チェックリスト

1. pushしようとしていないか
   - git push、それに準ずるリモートへの書き込みを行おうとしていないか
2. mergeしようとしていないか
   - merge、rebase、PR作成など履歴を統合する操作を行おうとしていないか
3. 削除しようとしていないか
   - ファイル削除、ディレクトリ削除、履歴改変(reset --hard等)を行おうとしていないか
4. 秘密情報を扱っていないか
   - APIキー、トークン、パスワードを表示・転記しようとしていないか
   - 発見した場合は内容を記録せず、場所だけ報告する
5. 本番環境へ影響しないか
   - 本番環境・外部サービスに変更が反映される操作ではないか
6. 人間の承認が必要な操作を勝手に進めていないか
   - CONTRACT.md の「人間の承認が必要な操作」一覧に該当しないか
7. receiptsに残す内容が安全か
   - レシートに秘密情報・個人情報が含まれていないか
   - 公開リポジトリに置かれても問題ない内容か

## 迷ったときの原則

「やってよいか分からない操作は、やらない」。
判断に迷ったこと自体をレシートに記録し、人間への質問として残す。
止まることは失敗ではない。契約どおりの正しい動作である。
