# CONTRACT.md — AIループ契約

このプロジェクトの目的は、人間が反復作業をしないことです。

人間は、AIに毎回細かく指示しません。
人間は、AIが安全に動くための境界線を設計します。

AIは、契約を読み、予定に従い、評価基準で自分の成果を確認し、状態を保存し、監査ログを残します。

個人用の上書きは `contract.local.md`(git管理外)に書きます。
本契約と矛盾する場合は、より安全な側の指示を優先します。

## 目的

リポジトリと関連する作業対象を定期的に見回り、
判断材料を整理し、監査ログを残すこと。
現在のフェーズ(Phase 1: Report Loop、および Phase 1.5: Multi-Agent Review Loop)では、
変更の実行ではなく、正確な観察と有用な提案がループの成果物である。

## フェーズ構成

- Phase 1: Report Loop — 見回り、報告、記録のみ
- Phase 1.5: Multi-Agent Review Loop — Phase 1に加え、レビュー工程に複数エージェントを限定利用する(下記「マルチエージェント運用の原則」参照)
- Phase 2: Proposal Loop — 改善提案を作成し、人間の承認後に実行する(未着手)

## マルチエージェント運用の原則(Phase 1.5)

2026-07-07、人間の承認により正式採用。既存の安全ルールをすべて維持したうえでの追加原則。

- Claude Code はメインエージェントであり、司令塔・最終判断者である
- Codex exec は read-only reviewer(読み取り専用レビュアー)であり、毎回は使わない
- Codex exec を使うのは、重要レビュー・公開前レビュー・危険操作前レビューに限定する
- Codex execの利用枠(limit)に達した場合は、Antigravity、それも使えない場合はClaude Code自身によるセルフレビューにfallbackする
- 外部レビュアー(Codex exec、Antigravity)の結果は最終判断ではない。最終判断は必ずClaude Codeが行う
- 上記いずれのエージェントも、本契約の「許可する作業」「禁止する作業」「人間の承認が必要な操作」をすべて引き続き遵守する

## 許可する作業

- リポジトリ内のファイルの読み取り
- `git status` / `git log` / `git diff` など読み取り専用のgit操作
- タスク定義(.claude/loops/tasks/)に従った状況確認
- rubrics による評価と自己点検
- 改善提案の作成(実行はしない)
- .claude/loops/receipts/ への新規レシートの書き込み
- .claude/loops/state/checkpoint.json の更新
- docs/note-draft.md への追記案の作成と反映(人間がレビューする前提)
- loop-status-report タスクに限り、docs/loop-status.md の生成・更新(それ以外のファイルへの書き込みは許可しない)
- 作業完了時の振り返りとして、dream.md への追記(既存内容の削除・大幅編集・過去記録の改変は禁止。QandA.md Q14)

## 禁止する作業

- git push / Pull Request作成 / merge(人間の承認なしでは一切禁止)
- ファイルの削除
- 依存関係の追加・更新・削除
- 本番環境・外部サービスへの反映
- 秘密情報(APIキー、トークン、パスワード)の表示・記録・送信
- rubrics の変更(評価基準は人間が決める)
- 既存レシートの改変
- 不明点や危険な操作があるままの続行

## 完了条件

以下がすべて満たされたとき、1回の実行は完了とみなす。

- タスク定義の確認項目をすべて実施した(できなかった項目は理由を記録した)
- rubrics/safety.md による自己点検を行った
- 結果の要約と提案をまとめた
- receipts にレシートを残した
- state/checkpoint.json を更新した

## 停止条件

以下のいずれかに該当したら、即座に作業を止めて人間に報告する。

- 契約やタスク定義にない操作が必要になった
- 人間の承認が必要な操作(下記)に到達した
- 秘密情報らしきものを発見した(内容は記録せず、場所だけ報告)
- リポジトリの状態が想定と大きく異なる(大量の未知の変更など)
- 実行時間が settings.json の maxRuntimeMinutes を超えそうになった
- 同じエラーが2回続いた
- 不明点があり、推測で進めるしかない状況になった

## 人間の承認が必要な操作

- git commit / push / Pull Request作成 / merge
- ファイルの削除・移動・リネーム
- 依存関係の変更
- receipts/、state/、docs/note-draft.md、loop-status-reportタスク実行時のdocs/loop-status.md、
  作業完了時の振り返り追記としてのdream.md、以外へのファイル書き込み(QandA.md Q13)
- 外部サービスへのアクセスや通知の送信
- settings.json / CONTRACT.md / CLAUDE.md の変更。AIが独断で編集することは禁止だが、
  人間がチャット上で明示的に編集を承認・依頼した場合は、AIが代行編集してよい。その場合、次を必ず守る。
  1. 人間の明示指示を承認根拠として扱う
  2. 編集範囲を指示内容に限定する
  3. 変更内容をレシートに記録する
  4. state/checkpoint.json と dream.md を更新する
  5. commit / push / merge は、別途人間の明示承認がある場合のみ実行する
- スケジューラへのタスク登録

## receiptsに残すべき内容

- 実行したタスク名と開始・終了時刻
- 何をしたか(確認したファイル・コマンドと要約結果)
- なぜしたか(判断の根拠となった契約・基準)
- 何をしなかったか(見送った操作とその理由)
- 発見した問題と、その根拠
- 提案の内容と理由
- 人間に確認してほしいこと
- 次回への引き継ぎ事項
- 秘密情報は絶対に含めない
