# contract.local.example.md — 個人用上書き設定の例

このファイルは例です。実際に使うときは `contract.local.md` という名前でコピーしてください。
`contract.local.md` は .gitignore によりgit管理されません(個人環境の設定だからです)。

```
cp contract.local.example.md contract.local.md
```

## 注意(最重要)

**このファイルにも contract.local.md にも、秘密情報を絶対に書かないでください。**
APIキー、トークン、パスワード、個人のメールアドレスなどは書いてはいけません。
git管理外でも、AIが読み込んでレシートに転記してしまう事故を防ぐためです。

## 対象リポジトリ名

- 例: mugen-loop
- 複数のリポジトリを見回りたい場合はここに列挙する(まずは1つから)

## 実行したい時刻

- daily-check: 例: 毎朝 9:00
- pr-hunter: 例: 平日朝 9:30
- note-experiment: 例: 毎晩 21:00
- メモ: 現段階では自動実行せず、この時刻に手動で起動する運用とする

## 確認したい対象

- 例: このリポジトリの git status と TODO
- 例: docs/note-draft.md の下書きの進み具合
- 将来: 特定のリポジトリのPR、特定ブログの下書きフォルダなど

## 通知先メモ

- 例: 結果は receipts/ を直接見る(通知連携はまだ使わない)
- 将来通知を使う場合も、送信内容は事前に人間が確認すること

## ローカル環境メモ

- 例: Windows環境。シェルスクリプトは Git Bash で実行する
- 例: エディタ、パス、タイムゾーンなど環境固有の事情をここに書く
